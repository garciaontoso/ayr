## Propósito

Las cartas trimestrales (módulo `cartas-sabios`) son **una** fuente de educación inversora, pero no la única. Los grandes inversores se construyen con:

1. **Libros** clásicos y modernos del value/quality investing
2. **Papers académicos** (factor investing, behavioral finance, dividend studies)
3. **Podcasts** (Invest Like the Best, We Study Billionaires, Capital Allocators)
4. **Cartas de fondos** (ya cubierto en cartas-sabios)
5. **Notas y articles** curadas (Bogleheads, Morningstar, Sequoia memos, Cobas papers)

Hoy esto vive disperso en bookmarks, Pocket, Kindle, papers descargados sueltos. Este módulo es el **knowledge management personal del inversor** — un solo lugar para tu biblioteca de educación, con tracking de progreso y conexiones cruzadas con tu cartera.

**Filosofía**: el conocimiento se valora cuando se usa. Una cita de Munger leída hace 5 años y olvidada vale lo mismo que no haberla leído nunca. Este módulo retiene tus aprendizajes y los conecta con tus decisiones reales.

---

## Filosofía

### Lo que NO es

- ❌ NO es Goodreads / Pocket / Kindle replacement (no compite)
- ❌ NO compra ni descarga libros
- ❌ NO es un curso online
- ❌ NO genera resúmenes de libros que no has leído (tentación AI a evitar)

### Lo que SÍ es

- ✅ **Inventario** curado de tu biblioteca de educación inversora
- ✅ **Tracking** de progreso (leyendo / leído / abandoned / re-read)
- ✅ **Quotes y notas** personales por item (capturadas mientras lees)
- ✅ **Cross-reference** con tickers/tesis ("este libro me hizo cambiar de opinión sobre X")
- ✅ **Reading queue** priorizado
- ✅ **Stats** (cuánto leíste este año, qué temas te faltan)
- ✅ **Recordatorios** suaves de re-lectura de clásicos

## Schema D1

```sql
-- Items de la biblioteca
CREATE TABLE library_items (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  type TEXT NOT NULL,                       -- 'book' | 'paper' | 'podcast' | 'article'
  title TEXT NOT NULL,
  author TEXT,
  year INTEGER,
  category TEXT,                             -- 'value classic' | 'behavioral' | 'macro' | etc
  language TEXT,
  source_url TEXT,
  isbn TEXT,                                 -- libros
  doi TEXT,                                  -- papers

  -- User state
  status TEXT DEFAULT 'queue',               -- 'queue' | 'reading' | 'read' | 'abandoned' | 're_reading'
  format TEXT,                               -- 'physical' | 'kindle' | 'pdf' | 'audiobook' | 'web'
  priority INTEGER DEFAULT 3,                -- 1-5 (5=must read next)
  tier TEXT,                                 -- 'S' | 'A' | 'B' | 'C' (curated importance)

  -- Reading tracking
  started_at TEXT,
  finished_at TEXT,
  abandoned_at TEXT,
  abandoned_reason TEXT,
  rating INTEGER,                            -- 1-5 stars (post-read)
  recommend_reread BOOLEAN,
  estimated_reading_hours REAL,

  -- Metadata
  added_at TEXT NOT NULL,
  notes_count INTEGER DEFAULT 0,
  quotes_count INTEGER DEFAULT 0
);
CREATE INDEX idx_li_status ON library_items(status);
CREATE INDEX idx_li_type ON library_items(type);
CREATE INDEX idx_li_priority ON library_items(priority DESC);

-- Notas personales por item
CREATE TABLE library_notes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id INTEGER NOT NULL,
  note_text TEXT NOT NULL,                   -- markdown
  page_or_chapter TEXT,                      -- 'p.45' | 'Ch.3' | '2:34' (podcast timestamp)
  themes_json TEXT,                          -- ['compounding', 'moat', 'patience']
  related_tickers_json TEXT,                 -- ['KO', 'PG'] tickers que esta nota me hace pensar
  related_thesis_id INTEGER,                 -- FK opcional a Módulo Proceso
  created_at TEXT NOT NULL,
  FOREIGN KEY (item_id) REFERENCES library_items(id)
);
CREATE INDEX idx_ln_item ON library_notes(item_id);

-- Quotes literales (para Quotes Wall, conecta con cartas-sabios)
CREATE TABLE library_quotes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id INTEGER NOT NULL,
  quote_original TEXT NOT NULL,
  language TEXT,
  page_or_location TEXT,
  themes_json TEXT,
  user_starred BOOLEAN DEFAULT 0,
  context_es TEXT,                           -- por qué la guardé
  added_at TEXT,
  FOREIGN KEY (item_id) REFERENCES library_items(id)
);

-- Conexiones a tickers/tesis (cross-reference)
CREATE TABLE library_ticker_connections (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id INTEGER NOT NULL,
  ticker TEXT NOT NULL,
  connection_type TEXT,                      -- 'inspired_buy' | 'inspired_sell' | 'changed_thesis' | 'mentioned' | 'general'
  connection_note TEXT,
  created_at TEXT,
  FOREIGN KEY (item_id) REFERENCES library_items(id)
);

-- Reading sessions (tracking de tiempo)
CREATE TABLE reading_sessions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id INTEGER NOT NULL,
  started_at TEXT,
  duration_minutes INTEGER,
  pages_read INTEGER,
  notes_added INTEGER,
  FOREIGN KEY (item_id) REFERENCES library_items(id)
);

-- Stats anuales
CREATE TABLE library_stats_yearly (
  year INTEGER PRIMARY KEY,
  books_completed INTEGER,
  papers_read INTEGER,
  podcasts_listened INTEGER,
  total_hours REAL,
  top_themes_json TEXT,
  generated_at TEXT
);
```

## Active suggestions

El módulo sugiere lecturas basado en:

1. **Gap temático**: tienes 6% en REITs pero 0 lecturas REIT-specific
2. **Ticker sin base de lectura**: tienes posiciones sin contexto educativo
3. **Re-read time**: clásicos no releídos en 7+ años (Buffett: relee Graham cada 5 años)
4. **Episodio podcast match**: nuevo episodio de Invest Like the Best con guest que tiene posiciones tuyas
5. **Carta de sabio reference**: Akre menciona libro X en su Q4 → sugerir si no lo tienes

Esto se puede extender al Discovery Engine como source: "Reading-driven discovery" — pero solo cuando un libro/podcast menciona un ticker concreto en contexto bullish.

## Integración con resto del sistema

### 1. Cartas Sabios
- Quotes Wall **unificado** con library_quotes (mismo botón, misma vista)
- Cuando un sabio menciona un libro en su carta → enlace a library con opción "añadir a queue"

### 2. Daily Briefing
- Si tienes algo "currently reading" desde hace >2 semanas sin avance → recordatorio suave
- Al completar un libro → "felicidades, lo añado al annual review"
- Sugerencias semanales (no diarias) en sección "💡 Reflexión"

### 3. Annual Review (Módulo Proceso)
- Sección "Lo que leíste este año" — top 5 libros, top temas, conexiones generadas
- Lecciones extraídas linkable a journal entries

### 4. Tesis (Módulo Proceso)
- Botón "Conectar lecturas a esta tesis" → muestra qué libros/notes apoyan la tesis
- Cuando creas una tesis nueva → "¿qué lectura te llevó a esta convicción?"

### 5. Journal de decisiones
- Campo nuevo "fuente de la idea" puede ser un library item
- Tracking: ¿qué libros generaron decisiones reales?

## Decisiones tomadas

| Decisión | Opción | Razón |
|----------|--------|-------|
| ¿Almacenar contenido de libros? | **NO, solo metadata + tus notas** | Copyright + scope creep |
| ¿Resúmenes auto-AI? | **NO** | Tentación peligrosa, mejor lectura propia |
| ¿Importar de Goodreads/Notion? | **CSV import en MVP** | Fricción mínima onboarding |
| ¿Reading sessions tracking? | **Sí, opcional** | Útil para gamificación suave |
| ¿Conectar a tesis/tickers? | **Sí, killer feature** | Diferencial real vs Goodreads |
| ¿Quotes Wall integrado con cartas?| **Sí, unificado** | Una sola colección de citas |
| ¿Stats anuales?| **Sí** | Annual review necesita esto |
| ¿Recommendations?| **Sí, basadas en gaps** | Contextualizadas a tu cartera |
| ¿Importance tier S/A/B?| **Sí, en seed** | Onboarding más claro |

## Riesgos y limitaciones

| Riesgo | Mitigación |
|--------|------------|
| User no captura notas mientras lee | UI minimal, mobile-friendly para captura rápida |
| Convertirse en otro Goodreads abandonado | Diferencial: conexión a tesis/cartera real |
| Suggestions repetitivas | Cooldown + variabilidad en sugerencias |
| Library queue infinita sin progreso | Recordatorio: "tienes 47 en queue, ¿priorizar 3?" |
| Quotes wall mezclado con cartas-sabios confunde | Filtros claros por origen (libro vs carta) |
| Tracking de tiempo se siente como gamification | Opcional, off por defecto |

## Decisiones aún pendientes

1. **¿Permitir uploads de PDFs?** → R2 storage similar a cartas-sabios. Podría ser útil para papers académicos. Decidir en Fase 2
2. **¿Sync con Kindle highlights?** → Amazon API es difícil. Opción: importar JSON de Readwise. Decidir Fase 5
3. **¿OCR de fotos de páginas físicas?** → tentador (foto a la página → quote auto-extraído). Necesita Cloudflare AI vision. Fase 6
4. **¿Compartir lista con compañeros?** → social features = scope creep. NO en MVP
5. **¿Reading challenges anuales?** → "lee 12 libros este año". Gamificación opcional, decidir en Fase 5
