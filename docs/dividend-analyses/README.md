# Dividend Analyses Archive

Long-form equity research reports for dividend-focused positions in the
A&R portfolio. Each file is an institutional-quality markdown report
generated by the Deep Dividend pipeline (Claude Opus 4.6 reading 7 years
of SEC filings and earnings transcripts).

## File naming convention

`{TICKER}-{YYYY-MM-DD}.md`

Examples:
- `KO-2026-04-10.md`  — Coca-Cola, generated April 10 2026
- `PG-2026-04-10.md`  — Procter & Gamble, same day
- `KO-2026-10-15.md`  — Coca-Cola, updated October 15 2026 (keeps history)

## File structure

Each report follows the institutional sell-side format (11 parts) plus
an AI-only insights layer. See the template in `_TEMPLATE.md` for the
section breakdown.

## How to generate

### Option A — Automated pipeline (paid)
Call the worker: `POST /api/earnings/archive/analyze { "ticker": "KO" }`
Uses Claude Opus API, costs ~$1.70/ticker, stores in D1 cache AND writes
the .md file to this folder (if the reporter script is enabled).

### Option B — Manual (free via Claude Code session)
The conversational Claude in a Claude Code session reads the archived
documents directly via `/api/earnings/archive/list` + `/get`, writes
the markdown report in-chat, and saves it via:
- `POST /api/earnings/archive/analyze/manual` (writes to the D1 cache)
- Direct filesystem write to this folder

Cost: $0 — uses the same conversational model, no API fees.

### Option C — Sync from cache
Run: `python3 scripts/sync-dividend-analyses.py`
This pulls all cached analyses from `/api/earnings/archive/analyses-cached`
and writes each one as a `.md` file in this folder.
